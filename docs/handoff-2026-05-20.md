## Current Status

Rowlabeler is live as a public deterministic Excel sequence label generator. The site is deployed through Vercel from GitHub `main`, and the production domain is managed in Cloudflare.

Accessible URLs:

```text
https://rowlabeler.com
https://www.rowlabeler.com
https://rowlabeler.vercel.app
```

Current production scope:

- Generate Excel 365 `SEQUENCE` formulas.
- Generate legacy Excel `ROW` formulas.
- Preview fixed values in browser.
- Download CSV preview output.
- Download TXT fixed-value output.
- Support English and Chinese pages.
- No login, payment, credits, AI, upload, or database-backed runtime feature is part of v1.

## Verification Commands

Run these before claiming a code or page-quality change is ready:

```powershell
pnpm exec tsx scripts/verify-excel-sequence-generator.ts
pnpm exec tsx scripts/verify-site-quality.ts
pnpm exec tsc --noEmit
```

Use this build path for local Vercel-style production verification on Windows:

```powershell
$env:VERCEL='1'
$env:CI='1'
$env:NEXT_TELEMETRY_DISABLED='1'
pnpm exec next build
```

Known caveats:

- Do not run `next dev` and `next build` together; they can fight over `.next`.
- Plain local standalone builds can hit Windows symlink `EPERM`; this is separate from Vercel build health.
- The template may log DB config warnings during static generation. That is acceptable while v1 remains a deterministic public tool without DB-backed runtime features.
- Real secrets must never be committed. Keep Cloudflare, Vercel, Search Console, Bing, and provider credentials out of docs and source files.

## Lessons To Preserve

Public copy lesson:

- Public pages must speak to the end visitor. Do not publish internal rationale such as "why this deserves a standalone site", "v1 scope", "SOP next step", "launch target", "planned output", or "source pain point". Those belong in docs, handoffs, or plans.

Launch lesson:

- Put the project on GitHub and connect Vercel early. After that, normal commits to `main` can deploy automatically and reduce manual drift.
- After custom domain setup, verify the live custom domain HTML, not only the Vercel preview URL.
- For Cloudflare DNS automation, use least-privilege scoped API tokens and never store tokens in the repo.

SEO lesson:

- For Google Search Console, use the full sitemap URL on domain properties.
- For Bing Webmaster, if typing a new sitemap in the UI is flaky, re-submit the existing sitemap row.
- After every sitemap change, submit IndexNow and refresh GSC/Bing sitemap status.

Long-tail lesson:

- Build small batches first. Two to four high-intent pages are better than a large thin-content dump.
- Each page must have locale JSON, `localeMessagesPaths` registration, sitemap entries, internal links, unique metadata, FAQ, and a CTA back to the tool.
- Keep `scripts/verify-site-quality.ts` updated whenever new page patterns are added.
